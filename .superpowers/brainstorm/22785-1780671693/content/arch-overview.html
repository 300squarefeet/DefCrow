Arsitektur Sistem — Pendekatan Mana yang Anda Inginkan?

Pilih bagaimana komponen sistem ini disusun secara keseluruhan

A

Monorepo: Web App + Rust Builder

Satu repositori berisi: backend API (Rust/Axum), frontend (React/Svelte), dan Rust loader template. Backend mengkompilasi loader secara langsung di server saat request masuk.

Pro

  • Satu deploy
  • Mudah dikembangkan lokal

Kontra

  • Kompilasi Rust di server butuh toolchain
  • Waktu generate lebih lama
B

Pisah: Builder Service + Web Frontend

Backend Python/Go sebagai API gateway. Rust builder berjalan sebagai microservice terpisah. Frontend Next.js/React standalone. Loader dikompilasi async dan hasilnya diunduh.

Pro

  • Skalabel
  • Builder bisa di-queue

Kontra

  • Lebih kompleks untuk deploy
C

Full Rust: Axum Backend + Template Engine + Frontend Static

Seluruh backend ditulis dalam Rust (Axum web framework). Loader template di-generate menggunakan Rust proc-macro / code generation. Frontend sebagai file statis yang disajikan dari Rust server.

Pro

  • Konsisten satu bahasa
  • Performa maksimal

Kontra

  • Kompleksitas tinggi di Rust
