Data Flow — Generate Loader

Request → Build → Download
User Web UI
  │  POST /api/generate  +  X-API-Key: <token>
  │  Body: { loader_type, features[], shellcode_b64, appdomain_config, pe_config }
  ▼
Axum Middleware (auth check)
  ▼
BuildQueue (tokio::spawn async task) → returns job_id immediately
  │
  ├─ 1. Decrypt & write shellcode → /tmp/job-{id}/shellcode.bin
  ├─ 2. Render appdomain.config.tera → /tmp/job-{id}/loader.config  (jika AppDomain)
  ├─ 3. cargo build --target x86_64-pc-windows-gnu
  │        --features "direct-syscall,sleep-encrypt,amsi-hwbp,..."
  │        -Z build-std
  ├─ 4. PE metadata inject (limelighter-rs)
  └─ 5. Mark job DONE, simpan artifact path

User polling GET /api/jobs/{id}  →  { status: "building" | "done" | "error" }
User GET /api/download/{id}      →  binary stream + Content-Disposition: attachment

AppDomain Feature — Detail

AppDomain Cross-Process Injection
Output yang di-generate:
  loader.dll        ← DLL yang di-inject ke proses target
  loader.config     ← AppDomain configuration (CLR version, runtime policy)

Teknik di dalam loader.dll:
  1. OpenProcess(target_pid)
  2. VirtualAllocEx → inject CLR hosting stub shellcode
  3. Stub memanggil CLRCreateInstance() → ICLRMetaHost
  4. GetRuntime("v4.0") → ICLRRuntimeInfo
  5. GetInterface(ICLRRuntimeHost) → Start()
  6. ExecuteInDefaultAppDomain(assembly_path, type, method, arg)

loader.config (Tera template):
  <configuration>
    <startup>
      <supportedRuntime version="{{ clr_version }}" />
    </startup>
    <appDomain name="{{ domain_name }}" />
  </configuration>

Web UI input untuk AppDomain:
  - Target process name / PID
  - CLR version (v2.0.50727 / v4.0.30319)
  - .NET assembly (upload file atau staged URL)
  - Entry point: Namespace.Class::Method

API Endpoints

Axum REST API
POST   /api/generate          ← kirim config, dapat job_id
GET    /api/jobs/:id           ← poll status: queued|building|done|error
GET    /api/download/:id       ← download artifact (binary stream)
DELETE /api/jobs/:id           ← hapus artifact dari server
GET    /api/health             ← liveness check (no auth required)

Header wajib (semua kecuali /health):
  X-API-Key: <token dari .env>
